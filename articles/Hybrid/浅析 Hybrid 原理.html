# 👉 浅析 Hybrid 实现原理

# Hybrid 技术起源

Hybrid 技术出现之前，移动端开发主要有两种类型：Web app 和原生 app，他们的特点对比如下：

  • 原生 App 功能强大，体验好，但开发迭代灵活性低，成本高；
  • Web App 相对原生 APP 功能少，体验稍差，但开发成本低，用户对迭代更新感知度相对低；

结合纯 APP 和 纯 Web 页面的优缺点，Hybrid 应用诞生了。作为一种混合开发的模式，Hybrid APP底层依赖于 Native 提供的容器（UIWebview），上层基于 Web 做业务开发，底层透明化、上层多多样化，这种场景非常有利于前端介入，非常适合业务快速迭代。简单来说，Hybrid 应用就是 Web 应用套了原生应用的外壳。Hybrid 技术：

  • 可以兼容多平台，可以调用手机系统提供的功能；
  • 它的性能、用户体验不如原生 app 但高于 纯 Web 页面；
  • 一定程度上，减少了原生 APP 体积和降低迭代更新频率。

# Hybrid 是如何实现的？

Hybrid 技术的实现主要依赖于以下两点：

  • file 和 schema协议
  • Webview

# file 和 schema协议

APP 中会经常用到三个协议：

  • file 协议：用于打开本地文件，优点是速度快；
  • https 协议：http 和 ssl/tls协议，这个协议需要发送网络请求，相对 file 协议会慢很多；
  • schema 协议：用于客户端和前端通讯的协议；
  • 微信部分 schema 协议列举如下：
    • weixin://dl/scan ：扫一扫
    • weixin://dl/moments ：打开朋友圈

# Webview

WebView 是移动端中的一个控件，也可以将其简单地理解成一个 App 内嵌的浏览器，它为 JS 运行提供了一个沙箱环境。WebView 能够加载并显示指定 URL 网页，拦截页面发出的请求、展示对话框等各种页面控制与交互功能，实现混合开发。客户端和前端的通信实现也依赖于 WebView 暴露的各种接口。

# Hybrid 如何加载页面资源

Webview 加载页面有两种方式：

  1. 加载网络页面，把 HTTP URL 传入 Webview 后触发发送网络请求加载对应资源；
  2. 加载本地页面，此处又细分两种：
  • 加载 assets 目录页面：不用进行资源请求即可访问本地离线文件，减少用户的数据流量；
  • 加载缓存到本地页面：用来做页面的离线缓存，将页面资源文件缓存存储到本地的文件目录下，下次访问同个 URL 时进行拦截，优先加载离线缓存，加快 Web 的资源加载速度（也叫“离线包”方案）。

如果 Webview 加载的是本地页面，则一般都是通过 file 协议来调用，如下：

详情见 WebView加载页面的两种方式——网络页面和本地页面：https://www.cnblogs.com/zxxiaoxia/p/6831085.html

加载本地资源肯定会比加载网络资源要快得多，但问题来了，若 Webview 中加载的页面资源需要经常迭代，肯定不能作为静态资源存放在客户端 asset 目录，不然每次就要升级 APP了，那如何让用户同步更新呢？客户端可以向服务端请求资源版本，只要检查到用户版本号和服务器版本号不一致时才去进行资源的请求。

# 预加载策略

客户端可以利用与服务端的交互，在打开APP时就向服务端请求，提前拉取资源最新版本号，并根据加载优先级配置来决定更新资源加载的时机。这种方式和发送请求网页资源不同之处在于，客户端可动态维护网页资源下发配置，决定客拉取新资源的时机，从而减少资源加载时长。比如：

  • 提前加载，打开 APP 后，触发APP向服务端请求资源版本情况，利用本地缓存资源版本号进行对比，版本号更新了的情况下，发出请求提前拉取新资源，替换本地缓存资源。
  • 按需加载，打开 APP 后，用户进入到网页入口所在页面停留 x 秒时，触发客户端向服务端请求资源版本情况，版本号更新的情况下，发出请求拉取新资源。
  • 异步加载，打开 APP 后，用户进入到网页后，再去向服务端请求资源版本情况，版本号更新的情况下，发出请求拉取新资源缓存至本地，待用户下次进入才会加载最新的版本资源。

# Hybrid 交互

JSB 原理浅析 Hybrid 的核心离开不开 Native 与 Web 的交互，而连接 Native 和 Web 的通信桥梁就是 JSBridge，JSBridge 通信又分为 Native -> Web和Web -> Native的两类通信方式。

  • Web -> Native：实现原理是 URL 协议拦截和注入式 API（客户端向 Webview 全局对象 Window 注入属性函数，Web 侧可以直接从全局对象中获取调用）。
  • Native -> Web：核心原理是将 JS 函数处理成字符串，传入至 Webview 进行解析编译。主要实现的方式是：loadUrl/evaluatingJavaScript。
